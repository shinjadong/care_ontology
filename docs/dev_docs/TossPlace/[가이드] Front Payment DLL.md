## 소개

이 문서에서는 토스플레이스의 Front와 타사 POS 기기들 간의 통신 프로토콜(DLL)을 정의합니다.

## API 목록

### 1\. 연결 및 연결 확인 API

프론트/프론트뷰 기기와 연결 하고 연결 상태를 확인하는 API

| API 이름 | API 설명 |
| --- | --- |
| GetDllVersion | DLL의 버전 확인 |
| ConnectToDevice | 기기와 연결 시도 |
| FindDevice | 입력된 시리얼 넘버로 연결 가능한 기기 찾기 |
| GetConnectionStatus | 연결 상태 확인 |
| SetStateChangeListener | 연결 끊김 등의 상태 변경 시 호출 되는 callback 등록 |
| SetConfig | DLL 내부 Config 값을 설정 합니다. (Config 목록 TBD: 로그 저장 여부, 로그 파일 경로 등등) |
| GetConfig | DLL 내부 Config 값을 확인 합니다. (Config 목록 TBD) |

#### GetDllVersion

const char \* GetDllVersion ();

| 리턴 값 | 설명 |
| --- | --- |
| ≠ NULL | DLL 버전 |

#### ConnectToDevice

typedef void (\_\_stdcall \* ConnectToDeviceCallback) (int status,const char \* message);int ConnectToDevice (const char \* target, ConnectionType type,int timeout, ConnectToDeviceCallback callback);

| 파라미터 | 유형 | 설명 |
| --- | --- | --- |
| target | const char\* | 대상 기기를 나타냅니다. CONNECTION\_SERIAL: (예시) ”COM3” CONNECTION\_IP: (예시) “192.168.10.100” 또는 “243F20~.local” |
| type | ConnectionType | 0 (CONNECTION\_SERIAL): 시리얼 1 (CONNECTION\_IP): IP 네트워크 |
| timeout | int | 연결 타임 아웃 (밀리초 단위) 시리얼통신일 경우 공백 |
| callback | ConnectToDeviceCallback | 연결 결과 callback (callback 이 null 인 경우 동기 방식으로 리턴) status: 1: 성공, < 0: 실패 message: 성공/실패 메세지 |

| 리턴 값 | 설명 |
| --- | --- |
| < 0 | 오류 (하단 오류 코드 참고) |
| 1 | 연결 성공 (동기 방식) |
| 2 | 연결 시작 요청 성공 (비동기 방식) |

#### GetConnectionStatus

int GetConnectionStatus (ConnectionType type);

| 파라미터 | 유형 | 설명 |
| --- | --- | --- |

| 리턴 값 | 설명 |
| --- | --- |

#### Config 목록

Config 는 이름과 값이 모두 string 타입이고 이름은 모두 소문자 입니다.

| 이름 | 기능 | 기본값 |
| --- | --- | --- |
| baudrate | SERIAL 연결 시 사용할 보드레이트 | 115200 |
| portnumber | TCP 포트 넘버 | 9000 |
| connect\_timeout | ConnectToDevice 호출 시 연결 타임 아웃 밀리초 | 5000 |

### 2\. POS ↔ Front 데이터 송수신 API

> 프론트와 POS 간 연동은 JSON 데이터를 사용합니다.

#### JSON 데이터 기본 구조

{ "version":"1.0","name":"commandName","data":{ } }

version

: JSON Data 구조를 구분하기 위한 버전

name

: JSON Data를 구분하는 Command 이름

data

: JSON Data
